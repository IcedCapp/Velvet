        To extend Velvet your constructor should call super with your preferred window size. All initialization code should be placed in init. Velvet's update loop is set to run as close to 60 frames per second as possible. This is the heartbeat of your program. The render loop is for redrawing the screen and is designed to run as frequently as possible. The VGraphics object is a wrapper for the java.awt Graphics2D object with some additional functionality. Finally the entry point for your program must call Velvet.start with an instance of your main class and a requested window title
